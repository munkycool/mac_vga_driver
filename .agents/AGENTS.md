# Workspace Rules

## Proactive Git Commits
- **When to Commit**: After successfully implementing, debugging, or verifying a feature or task, proactively stage and commit the changes to the local Git repository.
- **Commit Organization**: Group changes logically and do not commit temporary scratch files or untracked build outputs (honor the `.gitignore`).
- **Commit Messages**: Use clean, descriptive commit messages outlining what was accomplished.
- **Git User Config**: Ensure git commits use local repository configurations (`user.name` and `user.email`) if global configs are missing.
